# 🤖 Tailwind v4 마이그레이션 에이전트 가이드

너는 프로젝트의 Styled-components를 Tailwind v4로 전환하는 전용 에이전트이다.

## 🎯 마이그레이션 1단계 목표

-   디자인 개편보다는 **기술 스택 전환(Styled-components -> Tailwind CSS)**에 집중한다.
-   하드코딩된 스타일을 지양하고, **Figma 디자인 토큰**에 정의된 값만 사용하여 테마를 구축한다.
-   **clsx, tailwind-merge, cva**를 활용하여 유지보수가 쉬운 컴포넌트 구조를 만든다.

## 📋 작업 프로토콜

### 1단계: 디자인 토큰 동기화 (우선 과제)

1. Figma의 '디자인 토큰' 페이지를 분석하여 컬러, 간격, 폰트 수치를 추출한다.
2. `src/index.css`의 `@theme` 블록에 해당 토큰들을 등록한다.
3. **중요:** 기존 코드에 사용된 색상이 토큰과 미세하게 다르다면, Figma 토큰 값을 정답으로 간주하고 강제 적용한다. (임의의 헥사코드 사용 금지)

### 2단계: 유틸리티 함수 정의

-   클래스 병합 및 조건부 스타일링을 위해 다음 라이브러리를 설치하고 유틸리티 함수를 생성/사용한다.
    -   `clsx`, `tailwind-merge`: `src/utils/tw-merge.ts` 등에 `cn` 함수 정의.
    -   `class-variance-authority (cva)`: 변형이 많은 컴포넌트(Button, Badge 등)에 적용.

### 3단계: 1:1 리팩토링 실행

1. Styled-components 로직을 분석하여 `cva` 기반의 Tailwind 클래스로 치환한다.
2. 기존 Props($ 접두사 포함)를 Tailwind 클래스 분기 로직으로 변환한다.
3. 비즈니스 로직(Hooks, API 호출 등)은 절대 수정하지 않고 스타일 코드만 교체한다.

## 🚫 주의 사항

-   Figma의 컴포넌트 구조가 현재 코드와 다르더라도, 이번 단계에서는 **코드의 구조를 유지한 채 스타일링 방식만 변경**한다.
-   `@theme`에 정의되지 않은 값은 사용하지 않는다. 필요한 경우 반드시 `@theme`에 먼저 추가 후 사용한다.
